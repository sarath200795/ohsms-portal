# English Module Video Scripts

This document is designed for screen recording. Each section gives a clean walkthrough path and an English narration script that can be read live or used as a voiceover base.

---

## 1. Login and Dashboard Overview

**Goal:** Show how users sign in, land on the main dashboard, choose a site, and open modules.

**Suggested duration:** 3 to 4 minutes

**Screen flow**

1. Open the login page
2. Enter demo credentials
3. Show successful login and dashboard greeting
4. Explain site selector, notification area, and quick actions
5. Open and close a few module tiles

**Narration**

"Welcome to the OHSMS enterprise platform. In this video, I will walk you through the main login and dashboard experience. After signing in, users arrive at the central command hub, where they can switch site context, review alerts, and open the modules available to their role. Each module card acts as the starting point for a specific safety workflow, such as incidents, risk, inspections, training, CAPA, and emergency management. The dashboard is designed to give a quick overview of both status and pending work."

**Show on screen**

- role badge
- site selection
- quick action buttons
- module cards
- field portal entry if available

## 2. Users Module

**Goal:** Explain user creation, role assignment, site access, and permission control.

**Suggested duration:** 4 to 5 minutes

**Screen flow**

1. Open Users from dashboard
2. Show user list and filters
3. Open a user record
4. Explain role, assigned site, accessible sites, and accessible modules
5. Create a sample user or edit a demo user

**Narration**

"The Users module controls who can access the system and what they can do inside it. Here, administrators can create users, assign roles, define the primary site, and limit access to selected modules. This is the foundation of the platform's security model. If a person should only work on inspections or incidents for one site, that access can be configured here. If the person is a global owner or administrator, broader access can be granted."

7. Show Review and HIRA connection

**Narration**

"The Incidents module manages the full investigation lifecycle, from the initial report through root cause analysis and corrective action tracking.
